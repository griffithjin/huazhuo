|------|------|

| 合规性 | 满足金融、医疗、政务等合规要求 |
| 模型覆盖 | 全量通义千问系列、第三方模型 |
| 适用客户 | 中小企业、严肃应用、付费意愿强的客户 |

### 3.2 经济套餐 (寰卓闲散算力池)

**定位**: 价格敏感客户、批量推理、可容忍轻微延迟

**SLA**: 99.9% (允许秒级中断，自动重试)  
**定价**: 原价 **5-7折** (体验¥9.9 / 创作¥39.9 / 专业¥129)

| 特性 | 说明 |
|------|------|
| 极致性价比 | 推理成本降低30-50% |

## 4. 核心融合功能设计

### 4.1 智能调度网关 (Token Router)

```python
class TokenRouter:
    """
    寰卓 智能调度网关
    根据请求特征、客户等级、实时价格、SLA要求自动选择最优供给池
    """
    
    def route_request(self, request, api_key, user_tier):
        # 1. 请求特征分析
        req_type = self.classify_request(request)  # realtime / batch / async
        fault_tolerance = request.get('tolerance', 'low')  # low / medium / high
        
        # 2. 客户等级匹配
        if user_tier == 'enterprise' and req_type == 'realtime':
            # 企业实时请求 → 标准池 (ModelRouter)
            return self.model_router_pool.dispatch(request)
        
        elif user_tier == 'standard' or fault_tolerance == 'low':
            # 标准客户/低容错 → 标准池
            return self.model_router_pool.dispatch(request)
        
        elif user_tier == 'economy' or req_type == 'batch':
            # 经济客户/批量任务 → 经济池 (寰卓)
            return self.gongji_pool.dispatch(request)
        
        # 3. 实时比价优化 (企业定制客户)
        mr_cost = self.model_router_pool.quote(request)
        gj_cost = self.gongji_pool.quote(request)
        
        if gj_cost < mr_cost * 0.8 and fault_tolerance != 'low':
            # 寰卓池便宜20%以上，且容错允许 → 选择寰卓
            return self.gongji_pool.dispatch(request)
        else:
            return self.model_router_pool.dispatch(request)
    
    def handle_failover(self, request, failed_pool):
        """故障自动切换"""
        if failed_pool == 'gongji':
            # 寰卓池故障 → 降级到标准池
            return self.model_router_pool.dispatch(request, priority=True)
        elif failed_pool == 'model_router':
            # 标准池故障 → 尝试寰卓池 (可能成本更高但可用)
            return self.gongji_pool.dispatch(request, priority=True)
```

### 4.2 闲时算力反向托管 (客户侧变现)

**核心机制**: 寰卓客户可将自有/长租GPU的闲时时段接入寰卓调度池，赚取Token积分或现金分成

```
客户侧:
├── 已购买寰卓企业套餐 (如¥1999/月，含5000万Token)
├── 自有/长租 10张 RTX 4090 / A100
├── 闲时时段 (夜间/周末/业务低谷) GPU利用率 < 30%
└── 闲时GPU接入寰卓池 → 赚取Token积分

平台侧:
├── 将客户闲时GPU纳入寰卓调度池
├── 对外提供经济套餐服务
├── 收益分成: 客户60% + 寰卓 25% + 寰卓 15%
└── 客户 earned Token积分 可抵扣下次套餐购买

双赢效果:
├── 客户: 降低综合成本30-50% (闲置资产变现)
├── 寰卓: 扩展供给池，提升经济套餐毛利率
└── 寰卓: 增加调度节点密度，提升网络效应
```

**产品功能设计**:
1. **GPU接入向导**: 一键安装客户端，自动检测GPU型号/驱动/网络环境
2. **闲时设置**: 自定义闲置时段 (如 23:00-08:00, 周末全天)
3. **收益看板**: 实时显示GPU在线时长、贡献算力、赚取Token
4. **安全隔离**: 利用寰卓的Namespace/eBPF隔离技术，保障客户数据安全
5. **自动提现**: Token积分自动抵扣套餐续费，或满100元提现

### 4.3 绿色AI认证体系

**概念**: 利用寰卓的"算电协同"能力，将寰卓塑造为**绿色AI API平台**

| 绿色认证 | 机制 | 营销价值 |
|----------|------|----------|
| **绿电消纳** | 任务优先调度至绿电富余区域 | 每万次调用减少X kg碳排放 |

| **碳足迹追踪** | 为用户显示API调用的碳排放数据 | 企业ESG报告素材 |
| **绿色徽章** | 选择经济套餐获得"绿色AI"徽章 | 可用于客户产品宣传 |

## 7. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 寰卓池稳定性不足 | 经济套餐SLA不达标 | 智能降级: 寰卓故障→自动切ModelRouter；设置熔断阈值 |

| 价格竞争 | 寰卓池价格波动 | 动态定价算法: 设置保底价+市场价联动机制 |
