# 寰卓 技术团队开发说明

## 项目概述

寰卓（寰卓科技）- 中国版 OpenRouter，AI API Token 套餐化销售平台。

**核心定位**: 对接阿里云百炼 ModelRouter，将 AI 算力以 API Token 形式销售给 C 端用户和开发者。

---

## 商业模式

### 定价策略（基于真实成本）

1. **采购成本**: 阿里云百炼官方定价 × 寰卓科技折扣（月消费≥30万享 **68折**）
2. **用户售价**: 采购成本 × **2.0**（确保 **~50% 毛利率**）
3. **套餐折扣**: C 端购买额度享受阶梯优惠（66折-82折），平台仍保有 **25-40% 净利率**

## 系统架构

### 技术栈
- **前端**: React 18 + TypeScript + Ant Design + Vite
- **后端**: FastAPI + SQLAlchemy + PostgreSQL + Redis + Celery
- **部署**: Docker + Docker Compose / GitHub Pages (前端)
- **支付**: 支付宝企业支付
- **AI 供给**: 阿里云百炼 ModelRouter

### 核心数据流

```
C端用户 → 购买套餐/充值 → 获得额度
    ↓
生成 API Key (带额度限制)
    ↓
调用 API (选择模型)
    ↓
寰卓 后端
    ↓
阿里云百炼 ModelRouter (采购 Token)
    ↓
返回 AI 响应 + 扣除用户额度
```

## 关键技术决策

### 1. 额度扣减策略

**方案**: 预扣 + 后结算

1. 请求到达时，预扣 `max_tokens × 输出单价` + `input_tokens × 输入单价`
2. 实际响应返回后，根据真实 token 用量结算
3. 多退少补 (释放预扣额度，扣除实际用量)

**优点**: 防止并发超支
**缺点**: 需要 Redis 分布式锁

### 2. 额度原子性

使用 Redis Lua 脚本保证:
```lua
-- 扣减脚本
local balance = redis.call('GET', KEYS[1])
if not balance then return {-1, "key_not_found"} end
if tonumber(balance) < tonumber(ARGV[1]) then return {-2, "insufficient_balance"} end
redis.call('DECRBY', KEYS[1], ARGV[1])
redis.call('INCRBY', KEYS[2], ARGV[1])
return {1, "success"}
```

### 3. 并发控制

- 单 API Key: 默认 60 次/分钟 (可通过配置调整)
- 单 IP: 全局 100 次/分钟
- 使用 Redis + 滑动窗口算法

### 4. 模型路由

```python
# 伪代码
class ModelRouter:
    def route(self, model_id: str, request: dict):
        model = self.model_configs[model_id]
        
        # 根据 region 选择 endpoint
        if model.region == 'cn':
            endpoint = ALIYUN_CN_ENDPOINT
        elif model.region == 'global':
            endpoint = ALIYUN_GLOBAL_ENDPOINT
        
        # 转发请求
        response = self.forward(endpoint, request, model.provider_config)
        
        # 记录用量
        self.log_usage(request.api_key, model_id, response.tokens)
        
        # 扣减额度 (异步)
        self.charge_async(request.api_key, model_id, response.tokens)
        
        return response
```

## 重要提醒

### 1. 定价安全线

**企业包 ¥2799 / ¥5500 额度存在亏损风险！**

建议:
- 设置企业包最低消费限制
- 或限制企业包只能使用 Plus/Max 级别模型
- 或提高企业包售价至 ¥3199+ (确保 ≥15% 利润率)

### 2. API Key 安全

- 生产环境必须使用 HTTPS
- API Key 只存储哈希值，前端只显示前缀
- 支持 Key 级别额度限制和模型白名单
- 异常调用自动告警 + 自动禁用

### 3. 合规要求

- 用户实名认证 (手机号 + 姓名)
- 内容审核接入 (阿里云绿网)
- 敏感词过滤
- 调用日志保留 180 天

### 4. 百炼对接注意

- 百炼 Token 按官方价 × 折扣结算，非实时扣费
- 月度账单可能在次月 3-5 日生成
- 需要预留足够的阿里云账户余额
- 建议设置阿里云余额预警
